## 三、Live Demo（5分钟）

### Step 1：登录（15秒）

**操作：** 打开小程序，点击微信授权登录，进入首页。

**话术：**
> "登录后进入首页。黑塔作为导览形象出现在氛围区，同时顶部有'问黑塔'入口。"

**备用：** 如果登录网络慢，直接切到已登录账号截图。

---

### Step 2：进入 AI 页，展示 CRS 起点（30秒）

**操作：** 点击"问黑塔"进入 AI 对话页。

**话术：**
> "这个页面不是普通聊天页。它的核心是顶部的 **CRS 内联指示条**：显示当前模式（冷启动/混合/精准）和置信度进度条。置信度代表'黑塔对用户的了解程度'，越低说明越不了解，越高说明画像越完整。"

**展示重点：** CRS Badge（模式标签）+ 置信度进度条（显式/行为/对话三色）

---

### Step 3：冷启动 ASK 演示（40秒）

**操作：** 如果是新用户，页面会自动弹出 ASK 卡片；如果已是 user_id=1（有偏好），手动说一句"我想了解非遗，但不知道从哪开始"触发系统判断。

**话术：**
> "当系统置信度低于40时，黑塔不会直接给推荐，而是先问问题。这里叫 **ASK 策略**——系统从13个模板中选择一个高价值问题，例如'你最想了解哪类非遗？'
> 提问本身就是推荐质量的保障：如果问题问错了方向，后续所有推荐都是错的。"

**备选话术（如果已跳过冷启动）：**
> "新用户需要先走 ASK 流程才能建立偏好。这个账号已经完成冷启动，下面我们直接展示模式升级后的效果。"

---

### Step 4：模式升级展示（40秒）

**操作：** 点击一个 ASK 选项（如"传统工艺"），观察置信度变化和模式切换。

**话术：**
> "用户点击选项后，系统做了四件事：记录 ASK 日志、更新用户偏好、重新计算置信度、切换 CRS 模式。
> 置信度从原来的值上升，模式从 **cold_start（冷启动）** 升级为 **mixed（混合探索）**——此时系统会同时给推荐卡片和追问。"

**答辩亮点（可指给老师看）：**
- 置信度数字在变化（三维度条分别上升）
- 模式标签从"冷启动"变成"混合探索"
- 黑塔表情从"好奇"变成"思考"

---

### Step 5：推荐卡片与4层解释展示（60秒）★ 核心亮点

**操作：** 等待 AI 返回推荐卡片，点击第一张卡片的"展开解释"。

**话术：**
> "推荐结果不是只给一个标题。系统同时返回了 **4层推荐解释**："

| 层级 | 内容 | 展示时机 |
|------|------|---------|
| L1 用户可读 | 推荐理由文本（"因为你选择了传统工艺…"） | 默认展示 |
| L2 系统评分 | 综合评分 = 匹配分 + 新鲜度 - 惩罚项 | 点击"展开"后 |
| L3 策略上下文 | 当前CRS模式、用户画像来源 | 展开后 |
| L4 知识图谱 | 相似实体名、图谱推理路径 | 展开后 |

**话术续：**
> "这4层的逻辑是：普通用户只看 L1；老师想看依据可以看 L2/L3；答辩展示技术深度时可以打开 L4 看知识图谱路径。"

**强调点（必说）：**
> "与传统推荐系统的黑箱排序不同，这里的每张推荐卡都有**可追溯的打分明细**。匹配分来自画像匹配度，新鲜度惩罚来自内容去重，图谱分来自知识图谱相似度——这三个分数共同决定最终排名。"

---

### Step 6：点击内容卡，展示行为回流（30秒）

**操作：** 点击一张推荐内容卡，进入内容详情页（快速滑过，不详细讲内容），然后返回 AI 页。

**话术：**
> "用户点击内容这一行为，会被记录为 **impression → click → view** 三层行为日志。进入详情页算是 view，是更强的兴趣信号。"

**注：如果时间不够，这一步可以截图代替，但**行为回流**这个概念必须在第九步前建立。**

---

### Step 7：个人中心画像页 ★ P0-4 亮点（60秒）

**操作：** 进入个人中心页面。

**话术：**
> "这是个人中心，但它不是普通的'用户资料页'，而是 **系统理解用户的结果可视化**。整个页面分为4层叙事："

**L0 顶部 Hero 理解概览条（必指）：**
> "最上面是理解概览条——它告诉用户'黑塔现在对你的了解程度是多少'，以及当前处于哪个模式。"

**L1 画像信号全景（★ 重点展示）：**
> "中间是信号全景：左侧是置信度三元条，分别代表显式偏好（红色，用户主动选择）、隐式行为（蓝色，用户点击浏览）和对话语义（紫色，用户在AI对话中表达的）。右侧是 **偏好雷达图**——6个轴分别对应用户对戏曲、工艺、民俗、节俗、传统音乐、民间技艺的感兴趣程度。"

**L2 6维信号卡片（指给老师看一个）：**

**L3 我的记录（可跳过，简要带过）：**
> "最后是历史记录区，展示用户近期的行为轨迹。"

**关键话术（必须说）：**
> "这里的画像**不是静态的**——它随着 ASK 选择、点击行为、活动报名不断更新。冷启动用户进来时雷达图几乎空白；完成一轮推荐闭环后，雷达图会向用户真实兴趣方向扩展。"

### Step 8：CRS 调试接口（40秒）

**操作：** 在浏览器打开 `http://localhost:8000/api/v1/ai/crs/debug?user_id=1`（或提前准备截图）。

**话术：**
> "除了页面展示，系统还提供了完整的调试接口，可以查看**任意时刻的 CRS 决策链路**。"

**展示重点字段（指着页面）：**
```json
{
  "mode": "precision",              // 当前模式（已进入精准推荐）
  "confidence_score": 71.3,        // 综合置信度
  "score_explicit": 100.0,        // 显式偏好满分
  "score_implicit": 61.0,         // 行为信号
  "score_dialogue": 40.0,         // 对话语义
  "asked_attributes": ["category", "region", "scene"],  // 已采集的属性
  "last_ask_id": "A03",
  "recommend_cards_count": 2
}
```

**话术续：**
> "这个接口的逻辑完全对应前端的4层叙事。置信度计算公式是 **C = 0.40×Se + 0.35×Si + 0.25×Sd**——显式偏好权重最高，因为它最直接反映用户意图。"

---

### Step 9：CRS 统计接口（30秒）

**操作：** 打开 `/api/v1/ai/crs/stats`（或提前准备截图）。

**话术：**
> "最后，系统还支持**全局运营统计**，用于验证 CRS 闭环是否真实生效。"

**展示重点字段：**
- `ask_answer_rate`：ASK 回答率（回答/展示）
- `ask_skip_rate`：ASK 跳过率
- `avg_turns_to_converge`：冷启动平均收敛轮次
- `mode_distribution`：三种模式的会话分布
- `ctr`：推荐卡片点击率
- `behavior_implicit_delta`：行为对隐式分的影响量

**话术续：**
> "有了这个统计接口，答辩老师可以直接问：'你的冷启动收敛速度是多少？'——系统能给出数字，而不是只能说'我们觉得还不错'。"

## 四、技术亮点陈述（1分钟，5个点）

每个点不超过12秒，只说一句话+一个具体细节：

1. **AI数字人黑塔**：CSS纯代码实现，不依赖第三方素材，支持表情切换（好奇/思考/自信）和语音播报。

2. **CRS三维置信度**：综合置信度 = 0.40×显式 + 0.35×隐式 + 0.25×对话，三色条实时展示，无协同过滤。

3. **ASK-REC五策略状态机**：cold_start/ASK追问 → mixed/推荐+追问 → precision/精准推荐 → intent_driven/意图优先 → recovery_ask/纠偏重启。

4. **4层可解释推荐**：用户理由（L1）→ 系统评分（L2）→ 策略上下文（L3）→ 知识图谱推理（L4），每张推荐卡都有完整打分明细。

5. **行为闭环与画像可视化**：click/view/报名 → 隐式分更新 → 置信度提升 → 雷达图扩展，6维信号实时可见。

## Q&A 预测（提前准备）

| 问题 | 参考答案 |
|------|---------|
| 冷启动怎么解决的？ | ASK-REC五策略状态机，通过最小必要问题采集属性，置信度阈值（40/70）切换模式 |
| 为什么不用协同过滤？ | 协同过滤需要大量用户行为数据，冷启动场景下无效；CRS更适合非遗这类长尾内容 |
| 置信度公式怎么来的？ | 显式偏好权重最高（0.40）因为最直接，隐式行为次之（0.35），对话语义权重最低（0.25）因为噪声较大 |
| 知识图谱怎么参与推荐？ | 两个作用：①识别用户提问中的实体扩展候选；②把图谱路径作为L4解释字段附加到推荐卡 |
| 6维信号是什么？ | 偏好/浏览/AI提问/活动报名/社区互动/综合画像，从不同维度量化用户兴趣 |
| 系统规模能支撑答辩吗？ | 6条非遗内容+5个活动+12条知识库+55个图谱实体，足够演示CRS完整闭环；数据库可扩展到200条+无需改代码 |
| 为什么不用深度学习推荐？ | 毕设规模下数据量不足以训练有效模型（冷启动场景下协同过滤完全无效）；规则加权式推荐更容易解释和调试，也与论文可解释性主线一致 |
| 推荐解释4层有什么用？ | L1给普通用户看、L2/L3给有技术背景的人看、L4展示图谱推理路径；4层结构同时服务于用户体验和系统可信度 |
| 画像6维信号权重怎么定的？ | 显式0.40 > 隐式0.35 > 对话0.25——显式偏好最直接所以权重最高，对话噪声较大所以权重最低；三维度内部还有子权重（如click > expose） |
