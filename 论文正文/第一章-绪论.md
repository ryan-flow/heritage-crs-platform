# 第一章 绪论

## 1.1 研究背景

### 1.1.1 非物质文化遗产数字化传播的时代需求

非物质文化遗产（以下简称"非遗"）是各族人民世代相传的各种传统文化表现形式和文化空间，承载着民族的文化基因和精神血脉。截至2023年，中国列入联合国教科文组织非物质文化遗产名录的项目达43项，位居世界第一；国家级非物质文化遗产代表性项目达1557项，省级项目逾万项。然而，随着现代化进程加速，许多非遗项目面临传承人老龄化和受众流失的双重困境，保护形势严峻。

在此背景下，数字化成为非遗保护与传播的重要路径。杨红在研究非物质文化遗产数字化传播时指出，非遗数字化正经历从"以保存为目的的数字化记录"到"以传播为目的的数字化创作"的转型，传播重心从"专业领域"转向"大众视域"，传播方式从"单向展示"转向"数字化生存"【6】。这一转型呈现出三大趋势：

（1）**视频化与体验感。**短视频已成为非遗传播的核心形式。截至2022年6月，中国短视频用户达9.62亿，占网民总数的91.5%。非遗内容在抖音、快手等平台的传播数据显示，相关视频播放量年均增长超过150%。非遗体验正从博物馆的附属板块发展为独立成型的活动产品和体验场所，线上线下融合趋势明显。

（2）**要素开源化与数字化潜能。**非遗资源数字化后可不受时空限制触达受众，要素开源化使非遗可被衍生创作——动漫、游戏、产品设计等领域大量吸纳非遗元素。数字化为非遗提供了"数字化生存"的路径，使其从被保护的"遗产"变为可参与、可消费的"活态资源"。

（3）**数字化生存与虚拟社区传承。**非遗传播主体从"遗产持有者"扩展到"更大范围的社会公众"，形成了以兴趣驱动的非遗趣缘圈层。用户既是传播对象，也是传播主体，"兴趣传承"正在成为数字时代非遗传承的新模式。

与此同时，国家政策持续推动非遗数字化传播进程。《中华人民共和国非物质文化遗产法》第三条明确鼓励"运用现代信息技术和其他方式记录、保存非物质文化遗产"；《关于进一步加强非物质文化遗产保护工作的意见》提出"加大非物质文化遗产传播普及力度"的具体要求；《关于实施中华优秀传统文化传承发展工程的意见》进一步将非遗传播纳入文化强国战略。这些政策为非遗数字化平台的研发提供了制度保障和方向指引。

### 1.1.2 人工智能赋能文化传承的技术趋势

近年来，人工智能技术特别是大语言模型（Large Language Model, LLM）的突破性进展，为非遗文化传播提供了全新的技术可能。2023年以来，以ChatGPT、文心一言、豆包等为代表的大语言模型展现了强大的自然语言理解和生成能力，使"人机对话"从简单的关键词匹配进化为具有上下文理解的深度交互。

在文博领域，人工智能的应用已初见成效。康宁等报道的"辽小博"AI智慧导览系统，通过知识图谱与大模型融合微调技术，实现了博物馆场景下的智能问答和个性化导览，验证了AI技术在文化传承领域的可行性【7】。该系统的实践表明，在知识权威性要求极高的文博领域，大模型必须被结构化知识约束，采用"知识图谱融合微调"或"检索增强生成（RAG）"的方式，可以有效降低幻觉风险。

在推荐系统领域，大语言模型的应用同样引发范式变革。谢广明等指出，LLM在推荐系统中可以扮演语义增强器、生成器和策略控制器三种角色，为传统推荐系统注入语义理解和可解释性能力【3】。卡祖铭等进一步将LLM推荐系统分为判别式和生成式两大范式，指出"约束生成式LLM推荐"——即LLM在结构化知识约束下生成推荐结果和推荐理由——是兼顾灵活性与安全性的最优路径【4】。

### 1.1.3 微信小程序的平台优势

微信小程序作为轻量级移动应用形态，具有无需安装、即用即走、社交裂变等天然优势，特别适合文化传播类应用场景。截至2025年，微信月活跃用户超过13亿，小程序日活跃用户超过4.5亿。对于非遗传播而言，微信小程序的平台价值体现在三个方面：

（1）**触达门槛低。**用户无需下载安装，扫码或搜索即可使用，极大降低了非遗内容的接触成本，有利于触达更广泛的受众群体。

（2）**社交传播强。**微信的社交关系链为非遗内容提供了天然的传播渠道，用户可以将感兴趣的非遗内容分享到朋友圈、群聊，实现"以老带新"的社区传播效应。

（3）**开发成本低。**小程序原生开发框架成熟，组件丰富，适合中小型团队快速迭代，符合非遗数字化项目"小投入、快见效"的实际需求。

## 1.2 研究意义

### 1.2.1 理论意义

（1）**探索对话推荐系统在文化传承领域的应用范式。**赵梦媛等在对话推荐算法研究综述中指出，传统推荐系统存在"基于稀疏数据估计偏好""忽略上下文环境""假设用户清楚自身偏好"三大根本缺陷，对话推荐系统（CRS）通过多轮交互动态获取用户反馈、实时更新偏好，为解决上述问题提供了理论框架【2】。本研究将CRS理论引入非遗文化传播场景，通过AI数字人实现"系统提问—用户回答"的SAUR（System Ask, User Respond）交互范式，探索文化领域对话推荐的特殊性和方法论，丰富对话推荐系统的应用场景研究。

（2）**构建面向稀疏数据场景的多信号融合推荐模型。**非遗领域受众相对较小，用户行为数据天然稀疏，这是李小波等研究非遗推荐模型时指出的核心挑战【5】。本研究融合内容特征匹配、行为信号、场景感知、知识图谱推理和用户画像等多种信号源，构建规则加权式推荐策略，为数据稀疏场景下的推荐系统设计提供实践参考。

（3）**提出"约束生成式"AI导览的技术路径。**在非遗等知识权威性要求高的领域，大模型的幻觉风险不容忽视【3】【7】。本研究提出"本地知识库优先→联网搜索补充→大模型回退"的三级回退策略，结合内容治理体系（采集—清洗—审计—白名单回补），在保障知识准确性的前提下实现智能交互，为文化领域的AI应用提供安全可控的技术方案。

### 1.2.2 实践意义

（1）**解决非遗传播的"最后一公里"问题。**现有非遗数字平台多停留在单向展示阶段，用户被动浏览、缺乏互动，难以形成持续的文化参与。本研究构建的平台融合个性化推荐和AI数字人对话，主动向用户推送符合其兴趣的非遗内容，并通过对话引导用户深入了解，从"人找内容"变为"内容找人"。

（2）**缓解非遗推荐系统的冷启动困境。**新用户注册后缺乏历史行为数据，系统无法提供有效推荐，是当前非遗平台用户流失的主要原因之一。本研究通过AI数字人主动发起偏好探索对话，快速获取用户兴趣信号，构建初始用户画像，有效解决冷启动问题。

（3）**提升推荐的可解释性和用户信任度。**多数推荐系统只给出结果而不提供理由，用户难以理解为何被推荐某项非遗内容【1】。本研究通过知识图谱推理路径生成推荐理由，并由AI数字人以自然语言呈现，实现"推荐+解释"一体化，提升用户信任度和使用意愿。

（4）**为非遗数字化平台建设提供可复用的技术方案。**本研究的设计方案和实现代码可为其他文化类小程序（如博物馆导览、传统文化教育、地方文旅等）提供技术参考，降低同类项目的开发门槛。

## 1.3 国内外研究现状

### 1.3.1 推荐系统研究现状

推荐系统作为解决信息过载问题的关键技术，经历了从协同过滤到混合推荐再到深度学习推荐的演进历程。赵俊逸等系统梳理了协同过滤推荐算法的发展脉络，将其分为基于内存的方法（Memory-based，利用最近邻技术计算用户/物品相似度）和基于模型的方法（Model-based，通过矩阵分解、聚类、贝叶斯模型等机器学习方法建模）两大路线【1】。协同过滤的核心假设是"相似用户喜欢相似物品"，在电商、视频等领域取得了巨大成功，但面临数据稀疏性、冷启动、可扩展性、多样性不足和可解释性缺失等长期挑战。

针对上述问题，近年来推荐系统呈现出四个重要研究方向：（1）与图神经网络的结合，利用用户—物品交互的天然图结构建模高阶协同关系【1】【5】；（2）与知识图谱的结合，通过结构化知识增强物品间关联、提升推荐可解释性【1】；（3）交互式推荐系统，用户从被动接收转变为主动交互，通过对话或交互行为动态获取偏好【1】【2】；（4）推荐的可解释性，通过知识图谱推理或生成模型提供推荐理由【1】【3】。

在非遗推荐领域，李小波等提出了基于改进注意力机制的残差图神经网络（GNN-AR）推荐模型，针对非遗领域"用户行为数据极度稀缺""文化语义深度难以捕捉""用户深层兴趣偏好不易识别"三大特殊挑战，设计了图剪边器、残差多层图卷积、词向量注意力聚合和评分预测四层架构，在自建FeiYi非遗数据集上验证了模型的有效性【5】。该研究是当前最系统性地研究非遗推荐的工作，但未涉及场景感知、反馈闭环和智能导览能力。

### 1.3.2 对话推荐系统研究现状

对话推荐系统（Conversational Recommendation System, CRS）是推荐系统与对话系统的交叉领域，通过多轮交互实时获取用户反馈、动态更新偏好、打破信息不对称。赵梦媛等在对话推荐算法研究综述中系统梳理了CRS的研究框架，将CRS研究分为MII方向（关注用户交互形式的理解，涉及语音识别、自然语言语义理解等）和SP方向（关注对话策略的设计，涉及强化学习、知识图谱、图神经网络等）【2】。

CRS的核心范式为SAUR（System Ask, User Respond）——系统主动提问，用户回答，动态澄清偏好。当前CRS面临四大挑战：实时反馈获取、用户意图澄清、对话策略设计和推荐与对话的联合建模【2】。未来方向包括与知识图谱融合（为对话提供结构化知识底座）、多模态对话推荐（融合语音、图像、视频等多模态信息）和端到端对话策略学习（使用强化学习自动优化对话策略）。

对话推荐在电商、音乐、电影等领域已有初步应用，但在文化传播特别是非遗领域的应用研究尚属空白。非遗内容的特殊性——文化语义丰富、用户认知门槛高、兴趣偏好隐含——使得通用CRS方法难以直接迁移，需要结合非遗领域知识进行专门设计。

### 1.3.3 大语言模型与推荐系统的融合

大语言模型的突破性进展为推荐系统带来了新的技术范式。谢广明等从工业级应用视角综述了LLM在推荐系统中的三种角色：语义增强器（理解用户query和物品描述的深层语义）、生成器（生成推荐列表和推荐理由）、策略控制器（在对话推荐中决定交互策略），并指出LLM推荐面临数据偏见、评估标准不统一和幻觉风险三大挑战【3】。

卡祖铭等从技术范式视角将LLM推荐系统分为判别式LLM推荐（DLLM4Rec，LLM作为判别/排序模型对候选列表打分排序）和生成式LLM推荐（GLLM4Rec，LLM直接生成推荐结果或推荐理由）两大类【4】。其中，"约束生成式LLM推荐"——LLM在结构化知识约束下生成推荐内容——被认为是最适合知识权威性要求高领域的方向。该研究提出了一个关键洞察："利用LLM的力量优化推荐质量的关键在于：充分利用LLM高质量的文本特征表示以及广泛的外部知识库，以此为基础构建物品与用户之间的紧密联系。"【4】

然而，现有LLM推荐研究多聚焦于通用场景（电商、新闻等），在文化传承领域的应用研究尚处于起步阶段。如何在保障知识准确性的前提下发挥LLM的语义理解和生成能力，是本课题需要解决的关键问题。

### 1.3.4 AI数字人与文化导览

AI数字人是虚拟形象与智能对话的结合体，在文化导览领域具有独特优势。"辽小博"AI智慧导览系统是当前最成熟的文博领域AI导览实践，通过知识图谱与大模型融合微调技术，实现了博物馆场景下的智能问答、内容推荐、位置导航和个性化导览【7】。其技术架构（数据层—模型层—服务层）和问答处理流程（模板匹配→大模型生成→内容审核→输出）对本课题的AI数字人设计具有重要参考价值。

辽小博的实践也揭示了文博领域AI应用的核心矛盾：大模型具有强大的自然语言理解和生成能力，但容易产生幻觉和捏造不实信息；知识图谱具有知识准确和推理透明的优势，但缺乏生成能力和泛化能力【7】。该系统采用"知识图谱融合微调"策略解决这一矛盾，取得了良好效果。

然而，辽小博系统存在以下局限：（1）缺乏用户画像系统，个性化程度有限；（2）缺乏场景感知能力，推荐基于当前位置而非用户偏好与场景的综合；（3）缺乏反馈闭环，无法从用户行为中持续学习；（4）缺乏可视化数字人形象，交互形式较为单一。这些局限正是本课题的创新空间。

### 1.3.5 现有研究不足与本课题定位

综合以上分析，现有研究主要存在以下不足：

（1）**非遗推荐缺乏对话式交互。**现有非遗推荐系统（如GNN-AR模型）仍采用"一次性推荐"模式，用户被动接收推荐结果，无法通过对话澄清偏好或获取推荐解释【5】。

（2）**AI导览缺乏个性化推荐能力。**现有AI导览系统（如辽小博）具备智能问答功能，但推荐基于位置或热门排序，缺乏基于用户画像的个性化推荐【7】。

（3）**大模型在文化领域的应用缺乏系统性安全约束。**现有研究多关注LLM的推荐能力，对幻觉风险的系统性防控机制讨论不足【3】【4】。

（4）**缺乏面向非遗场景的一体化平台。**现有工作或侧重推荐算法、或侧重AI对话、或侧重数字展示，尚未有将个性化推荐、AI数字人对话、知识图谱增强、社区互动等功能整合于一体的非遗传播平台。

本课题定位为：面向非遗文化传播场景，构建融合多源内容治理、个性化推荐与AI数字人导览的微信小程序平台，实现"内容展示—智能问答—个性化推荐—活动参与—社区互动"的一体化服务闭环，重点解决非遗推荐的冷启动问题、推荐可解释性问题和AI导览的知识安全性问题。

## 1.4 论文组织结构

本文共分为七章，各章内容安排如下：

**第一章 绪论。**阐述研究背景、研究意义、国内外研究现状和论文组织结构，明确本课题的研究问题和创新点。

**第二章 相关技术与理论基础。**介绍本系统涉及且已真实落地的关键技术，包括微信小程序原生开发、FastAPI 后端服务、SQLite 数据存储、AI 数字人问答链路、CRS 对话推荐机制、轻量推荐算法与知识图谱增强方式等。

**第三章 系统分析。**从非遗数字化传播的现实需求出发，进行功能需求分析、非功能需求分析和可行性分析，明确系统建设目标与约束条件。

**第四章 系统设计。**在需求分析基础上进行系统整体架构设计、AI 数字人设计、CRS 推荐机制设计、数据库设计和接口设计。

**第五章 系统实现。**详细阐述系统主要功能模块的实现过程，包括前后端实现、AI 服务实现、推荐引擎实现、知识图谱增强和行为闭环实现。

**第六章 系统测试与实验分析。**对系统的功能、性能和兼容性进行测试，并对推荐效果、CRS 冷启动改善、知识图谱增强和行为回流闭环进行实验分析。

**第七章 结论与展望。**总结本研究的主要工作，并对系统后续完善方向进行展望。
