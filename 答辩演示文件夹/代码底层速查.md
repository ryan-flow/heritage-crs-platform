## 三个核心公式

### 1. 置信度公式
```
C = 0.40 × S_explicit + 0.35 × S_implicit + 0.25 × S_dialogue

S_explicit: 用户主动设置的偏好，每个词+30(上限60)
             + ASK回答每次+10(上限30)
S_implicit: 浏览记录 click+1.2 view+1.6 expose+0.35(上限40)
             + 活动报名+15(上限30) + 社区互动+8(上限30)
S_dialogue: AI提问+4(上限40) + ASK有效回答+12(上限60)
```
📍 `recommendation_service.py` L1101-L1107, L1225-L1249

### 2. 模式切换阈值
```
C < 28   → cold_start  (纯提问)
28 ≤ C < 62 → mixed   (边推荐边追问)
C ≥ 62  → precision (精准推荐，停止追问)
```
📍 `recommendation_service.py` L1106-L1107

### 3. 推荐排序公式
```
match_score = heritage × 1.4 + entity_bonus + scene × 0.8
            + quality × 2.2 + featured_bonus + pop_bonus
final_score = match_score + novelty_score - seen_penalty

已看惩罚: content-12, event-10, topic-8
```
📍 `recommendation_service.py` L711-L726, L51

## 四张速查表

### 表1：CRS决策引擎怎么选策略

| 条件 | 策略 | 行为 |
|------|------|------|
| 用户说"推荐/适合/先看什么" | intent_driven_rec | 直接推荐，无视置信度 |
| 连续3轮无互动 + C<28 | recovery_ask | 换方式重新提问(R01) |
| C<28 + 单数轮 | cold_start_ask | 顺序问A01→A02→A03→A04→A05 |
| C<28 + 双数轮 | mixed(穿插) | 给点推荐避免疲劳 |
| 28≤C<62 | mixed | 推荐+追问同时进行 |
| C≥62 | precision | 纯推荐，停止追问 |

📍 `crs/decision.py` L26-L72

### 表2：一条问题怎么变成偏好

| 来源 | 例子 | 映射结果 |
|------|------|---------|
| 用户选ASK选项"传统工艺" | `mappings.py:OPTION_TO_PREFERENCE` | → heritage += "工艺" |
| 用户自由输入"苏绣" | `mappings.py:QUESTION_HERITAGE_MAP` | → "绣" → heritage += "工艺" |
| KB命中chapter="传统工艺" | `mappings.py:KB_CHAPTER_TO_PREFERENCE` | → heritage += "工艺" |
| KG识别实体"苏绣" | `mappings.py:KG_ENTITY_TO_CATEGORY` | → heritage += "工艺" |

### 表3：五级回退怎么选回答

| 级别 | KB状态 | KB置信度 | 用谁回答 |
|------|--------|---------|---------|
| L1 | 命中 | ≥0.8 | KB原文 + 豆包润色 |
| L2 | 命中 | 0.45~0.8 | 豆包参考KB回答 |
| L3 | 未命中 | — | 豆包直接生成 |
| L4 | 豆包失败 | — | 先联网搜索，再豆包润色 |
| L5 | 全失败 | — | 硬编码兜底回复 |

### 表4：6维信号怎么给推荐打分

| 信号 | 数据来源 | 权重 |
|------|---------|------|
| 偏好画像 | users.preferred_heritage_types | 3.0 |
| 当前问题 | 从问题文本提取关键词 | 3.0(普通词)/6.0(实体词) |
| 浏览记录 | recommend_logs.click/view | 1.2/1.6 |
| AI提问 | ai_qa_logs | 1.1 |
| 活动报名 | activity_registrations | 1.4/1.5 |
| 社区互动 | discussion_likes/favorites | 1.2/1.3 |
| 兜底 | heritage为空时 | 戏曲1.6 + 工艺1.6 |

## 万一被问到的问题，三句话回答模板

**"为什么不直接用大模型做推荐？"**
→ 大模型推理成本高、延迟大、可能编造。我用本地知识库约束输出，推荐算法用规则加权——结果可解释、成本可控。

**"知识图谱怎么建的？"**
→ SQLite两张表(entities+triples)，59实体85条关系，BFS做最短路径和扩展推荐。轻量但够用。

**"CRS和普通聊天机器人有什么区别？"**
→ 聊天机器人只管回答，CRS在回答的同时：①同步用户偏好到画像，②计算置信度判断了解程度，③根据置信度决定继续提问还是直接推荐，④把推荐结果带解释地展示出来。

**"你要演示什么？"**
→ 登录→AI页→问5个问题演示CRS冷启动到精准的完整过程→回首页看推荐变化→展示画像页。整个过程约5分钟。

---

> 核心就一句话：**用户问 → 搜知识库 + 豆包回答 → CRS决策要不要追问 → 6维信号算推荐 → 返回回答+推荐+解释**。所有代码都围绕这条链路。
